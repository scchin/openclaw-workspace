# 我要去哪（where-to-go）技能

## 何時啟動

使用者在同一句話中提到以下任一關鍵字時，自動啟動：

| 餐飲時段 | 設施 | 地點 | 食物類型 |
|---------|------|------|---------|
| 早餐、午餐、晚餐、宵夜、下午茶、點心 | 廁所、加油站、停車場 | 美術館、博物館、圖書館、公園、景點、學校、旅館、民宿 | 火鍋、壽司、合菜、快炒、素食、港式、飲茶、燒肉、牛排、披薩、義大利麵、咖哩、拉麵、早午餐、咖啡、甜點、海鮮、日本料理、韓式、泰式、吃到飽…等 |

> 同一句話多個關鍵字 → 以**第一個**出現的為主。

---

## 完整流程

```
Step 1 → 解析使用者輸入（關鍵字、圓心、半徑）
Step 2 → 搜尋候選地點（半徑自動擴張）
Step 3 → 過濾 + 排序（評分 ≥ 門檻，由近到遠）
Step 4 → 【食物關鍵字】評論過濾（6個月內）
Step 5 → 取評分最高的 5 家作為候選
Step 6 → 重新按距離由近到遠排列（★★★ 2026-03-22 新增）
Step 7 → 呼叫『Google Places 地點查詢技能』查詢每個地點完整資訊
Step 8 → 依距離順序輸出所有結果
```

---

## Step 1｜解析使用者輸入

檢查使用者是否提供以下資訊：

| 資訊 | 說明 |
|------|------|
| 所在地 | 地址、地點名稱、經緯度 |
| 範圍半徑 | 公尺數、開車分鐘數、或走路分鐘數 |

**半徑換算**：
- `500m` 或 `500` → 500 公尺
- `1.5km` → 1500 公尺
- `開車 10 分` → 500 × 10 = 5000 公尺
- `走路 15 分` → 80 × 15 = 1200 公尺

**預設值**（使用者未提供任何地點時）：
- 圓心：`24.18588146738243, 120.66765935832942`（我家）
- 半徑：`1000m`

---

## Step 2｜搜尋候選地點

```bash
goplaces search "<關鍵字>" \
  --lat <緯度> --lng <經度> --radius-m <半徑> \
  --open-now --min-rating 4.5 --limit 30 --json
```

**預設參數**：
- `--open-now`：只看現在有開的
- `--min-rating 4.5`：評分 4.5 以上
- `--limit 30`：取 30 筆候選（食物過濾會淘汰部分）

**半徑自動擴張**（找不到 ≥ 5 家時）：
```
1000 → 1500 → 2500 → 5000 → 10000 → 20000 → 50000 m
```

---

## Step 3｜過濾 + 排序

1. 移除 `CLOSED_PERMANENTLY`（永久停業）
2. 評分低於門檻 → 移除
3. **由近到遠**排序（Haversine 直線距離）
4. 取前 **5** 名

---

## Step 4｜食物關鍵字過濾（★★★ 2026-03-22 新增）

當使用者輸入包含**食物關鍵字**（如「火鍋」「素食」「港式」）時：

1. 對每個候選地點呼叫 **Google Places API** 取得 6 個月內評論
2. 掃描範圍：
   - 店家名稱、類型、說明
   - 熱門品項（popular items）
   - **6 個月內評論原文**（原文優於翻譯）
3. 比對方式：
   - 直接關鍵字（火鍋 → `火鍋`、`涮涮鍋`、`麻辣鍋`、`鍋物`）
   - 別名擴充（如「港式」包含`茶餐廳`、`燒臘`、`腸粉`、`叉燒`）
4. **只有提到該食物的店家才入選**，其餘替補至數量補足
5. 若替補後仍不足 5 家，放寬至全部過濾後結果

---

## Step 5｜『Google Places 地點查詢技能』完整查詢（★★★ 2026-03-22 更新）

**自動對每個入選地點執行** `query.py full <place_id>`，不需詢問用戶：

- 地址（中文化）、電話、經緯度
- 今日營業狀態 + 距離關門時間
- **每人消費（CDP 動態讀取）**
- **價格分布柱狀圖（CDP 預設功能）**
- 服務類型（內用/外帶/外送）
- 熱門品項
- 6 個月內用戶價格反饋（`+49$`、`$150`、`CP值200` 等）
- Google Maps 連結

---

## Step 6｜最終輸出

一次輸出所有 5 家的完整資訊，格式：
- 🥇 🥈 🥉 4️⃣ 5️⃣ 排名
- 店名｜評分｜符合食物標籤（如有）
- 直線距離 + 估算時間
- Google Maps 連結
- 完整 Google Places 查詢結果（address / phone / 營業狀態 / 每人消費 / 分布圖 / 網友心得）

## 執行指令

### run.py — 完整流程（推薦使用）

整合 Step 2→3→4→5→6，自動搜尋 → 過濾 → 食物比對 → 完整查詢 → 輸出。

```bash
# 餐飲時段
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 我想吃早餐

# 食物類型（自動過濾評論）
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 火鍋
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 素食 4.5分
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 港式 1km內

# 設施 / 地點
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 找停車場 開車5分
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 附近美術館

# 混合條件
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 火鍋 24.1858,120.6677 2km 評分4.5以上
```

### search.py — 快速列表（只用 Step 2–3，無詳細查詢）

```bash
python3 ~/.openclaw/skills/where-to-go/scripts/search.py 早餐 24.18588146738243 120.66765935832942 1000
```

## 注意事項

1. **食物關鍵字智慧比對**：Google 評論中只有明確提到相關食物的店家才入選，並支援常見別名（如「素食」含「蔬食」、「Vegan」）
2. **當下有開**：`--open-now` 過濾，並在輸出中標注營業狀態
3. **評分門檻**：預設 4.5，使用者可調整（如「4.0 分以上也可以」）
4. **數量預設**：5 家，使用者可調整（如「給我 10 家」）
5. **半徑擴張**：最多擴張到 50km，以找到足夠店家為主
6. **完整查詢費時**：5 家約 1–2 分鐘（Google Places API + CDP），已自動執行不需詢問
7. **電話 / 營業時間**：goplaces search 結果中電話不一定有，完整查詢會補足

## 依賴

- `goplaces` CLI（`brew install steipete/tap/goplaces`）
- `GOOGLE_PLACES_API_KEY` 環境變數
- CDP 瀏覽器（`browser(action="start", profile="openclaw")`）→ 由『Google Places 地點查詢技能』內建使用
