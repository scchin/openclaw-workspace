# 我要去哪（where-to-go）技能

---

## ⚠️ 最高優先原則（強制遵守，無例外）

### １．輸出時機：100%
每一次使用者說「我要吃 XX」或任何餐飲需求，**立即執行** `run.py`，不詢問、不確認、不等待。

### ２．輸出完整性：100% — 不得濃縮、不得摘要、不得省略
**每一個回覆都必須是 `run.py` 的完整原始輸出，不得做任何文字上的濃縮、摘要，省略或選擇性呈現。**

這是鐵律，沒有「方便閱讀」或「自行判斷」這種選項。
使用者的需求明確，系統就完整執行並完整呈現。

### ３．輸出格式（固定模板，每次完全一致）

```
🥇 ════════════════════════════════════════════════════
🏪 店名：{店名}｜⭐ 評分：{評分}（{評論則數}則）｜🍴 符合：{食物關鍵字}
📏 直線距離：{距離}m（🚗 {開車分鐘}分 🚶 {走路分鐘}分）
📍 地址：{完整地址}
📍 經緯度：{緯度}, {經度}
📞 電話：{電話}
💵 每人消費：{每人消費區間}
📌 服務：{服務項目}
🔥 特色菜色：{菜色名}：{摘要}
📝 網友心得：{作者}：{內容}
🚦 狀態：{✅/❌} 營業中/休息中（今日 {營業時段}，{距離關門/開門時間}）
🌐 網站：{網址}
🔗 Google Maps：{Google Maps 連結}
```

**12 欄位剛性要求（每一個店家區塊都必須包含）：**
1. 🏪 店名（含評分、評論數、食物符合標籤）
2. 📏 直線距離
3. 📍 地址（中文化完整地址，含郵遞區號）
4. 📍 經緯度
5. 📞 電話
6. 💵 每人消費（無則顯示「店家未提供」）
7. 📌 服務（無則省略該行）
8. 🔥 特色菜色（無則顯示「無（評論中未抓獲特定菜色）」）
9. 📝 網友心得（無則顯示「無（近期無評論）」）
10. 🚦 營業狀態
11. 🌐 網站（無則顯示「無」）
12. 🔗 Google Maps 連結

### ４．絕對禁止事項
- ❌ 不得只給「濃縮摘要」或「精選推薦」
- ❌ 不得只給 3 家或只給前 2 名
- ❌ 不得跳過「無」欄位（`無（評論中未抓獲...）` 也是完整輸出）
- ❌ 不得在回覆中摻入我自己的補充說明（除非使用者明確提問）

### ５．輸出後再說
完成 `run.py` 原始輸出後，可以視情況加一句「想吃哪間？」之類的結語，但**不得修改、濃縮或省略原始輸出本身**。

## 完整流程

```
Step 1 → 解析使用者輸入（關鍵字、圓心、半徑）
Step 2 → 搜尋候選地點（半徑自動擴張）
Step 3 → 過濾 + 排序（評分 ≥ 門檻，由近到遠）
Step 4 → 【食物關鍵字】評論過濾（6個月內）
Step 5 → 取評分最高的 5 家作為候選
Step 6 → 重新按距離由近到遠排列（★★★ 2026-03-22 新增）
Step 7 → 呼叫『Google Places 地點查詢技能』查詢每個地點完整資訊
Step 8 → 依距離順序輸出所有結果
```

---

## Step 1｜解析使用者輸入

檢查使用者是否提供以下資訊：

| 資訊 | 說明 |
|------|------|
| 所在地 | 地址、地點名稱、經緯度 |
| 範圍半徑 | 公尺數、開車分鐘數、或走路分鐘數 |

**半徑換算**：
- `500m` 或 `500` → 500 公尺
- `1.5km` → 1500 公尺
- `開車 10 分` → 500 × 10 = 5000 公尺
- `走路 15 分` → 80 × 15 = 1200 公尺

**預設值**（使用者未提供任何地點時）：
- 圓心：`24.18588146738243, 120.66765935832942`（我家）
- 半徑：`1000m`

---

## Step 2｜搜尋候選地點

```bash
goplaces search "<關鍵字>" \
  --lat <緯度> --lng <經度> --radius-m <半徑> \
  --open-now --min-rating 4.5 --limit 30 --json
```

**預設參數**：
- `--open-now`：只看現在有開的
- `--min-rating 4.5`：評分 4.5 以上
- `--limit 30`：取 30 筆候選（食物過濾會淘汰部分）

**半徑自動擴張**（找不到 ≥ 5 家時）：
```
1000 → 1500 → 2500 → 5000 → 10000 → 20000 → 50000 m
```

---

## Step 3｜過濾 + 排序

1. 移除 `CLOSED_PERMANENTLY`（永久停業）
2. 評分低於門檻 → 移除
3. **由近到遠**排序（Haversine 直線距離）
4. 取前 **5** 名

---

## Step 4｜食物關鍵字過濾（★★★ 2026-03-22 新增）

當使用者輸入包含**食物關鍵字**（如「火鍋」「素食」「港式」）時：

1. 對每個候選地點呼叫 **Google Places API** 取得 6 個月內評論
2. 掃描範圍：
   - 店家名稱、類型、說明
   - 熱門品項（popular items）
   - **6 個月內評論原文**（原文優於翻譯）
3. 比對方式：
   - 直接關鍵字（火鍋 → `火鍋`、`涮涮鍋`、`麻辣鍋`、`鍋物`）
   - 別名擴充（如「港式」包含`茶餐廳`、`燒臘`、`腸粉`、`叉燒`）
4. **只有提到該食物的店家才入選**，其餘替補至數量補足
5. 若替補後仍不足 5 家，放寬至全部過濾後結果

---

## Step 5｜『Google Places 地點查詢技能』完整查詢（★★★ 2026-03-22 更新）

**⚠️ 強制執行：每一個入選地點都必須執行 `query.py full <place_id>`，直接輸出完整內容，不跳過任何欄位。**

**執行方式：**
```bash
python3 ~/.openclaw/skills/google-places/scripts/query.py full <place_id>
```

**⚠️ 嚴禁事項：**
- 不得過濾掉 `query.py full` 的任何實質輸出行（地址、電話、評分、金額、評論等）
- 不得只輸出基本列表就結束，必須包含上述所有欄位
- 不得在輸出前詢問用戶「要不要查詳細」，預設就是全部查完輸出

---

## Step 6｜最終輸出

一次輸出所有 5 家的完整資訊，每一家都必須包含上述 12 欄位。

## 執行指令

### run.py — 完整流程（推薦使用）

整合 Step 2→3→4→5→6，自動搜尋 → 過濾 → 食物比對 → 完整查詢 → 輸出。

```bash
# 餐飲時段
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 我想吃早餐

# 食物類型（自動過濾評論）
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 火鍋

python3 ~/.openclaw/skills/where-to-go/scripts/run.py 港式 1km內

# 混合條件
python3 ~/.openclaw/skills/where-to-go/scripts/run.py 火鍋 24.1858,120.6677 2km 評分4.5以上
```

## 注意事項

1. **食物關鍵字智慧比對**：Google 評論中只有明確提到相關食物的店家才入選，並支援常見別名（如「素食」含「蔬食」、「Vegan」）
2. **當下有開**：`--open-now` 過濾，並在輸出中標注營業狀態
3. **評分門檻**：預設 4.5，使用者可調整（如「4.0 分以上也可以」）
4. **數量預設**：5 家，使用者可調整（如「給我 10 家」）
5. **半徑擴張**：最多擴張到 50km，以找到足夠店家為主
6. **完整查詢費時**：5 家約 1–2 分鐘（Google Places API + CDP），已自動執行不需詢問
7. **電話 / 營業時間**：goplaces search 結果中電話不一定有，完整查詢會補足
