# 🛡️ 從失效到永恆：OpenClaw $\text{LaTeX}$ 物理攔截與「雙層鎖定」架構演進指南

## 前言：對抗「隨機性」的工程戰鬥
在 OpenClaw 的演進過程中，$\text{LaTeX}$ 符號（尤其是 $\rightarrow$）的污染被視為系統邏輯崩潰的標誌。我們經歷了從「單純修補」到「系統級防禦」的過程。本指南詳細記錄了攔截機制如何從一個脆弱的插件演變為一套確定性的「雙層鎖定」架構。

---

## 第一章：初始實作與「靜默失效」事件

### 1.1 初始方案：Bundle 物理注入
最初，我們採取直接修改編譯後 JS 檔案 (`server.impl-*.js`) 的方案。透過注入 `_purify` 遞迴函數，在 `JSON.stringify` 之前強行攔截所有字串並替換符號。

### 1.2 失效現象
在一次系統更新後，原本生效的攔截機制在毫秒之間完全失效。輸出內容重新出現大量 `$\rightarrow$` 符號。

### 1.3 根源分析：衍生資產 (Derived Assets) 的覆蓋
經過專家診斷，失效原因並非邏輯錯誤，而是**物理覆蓋**：
- **機制**：編 compiled 後的 JS 檔案屬於衍生資產。每當 OpenClaw 進行版本更新、依賴更新或重新部署時，系統會用原始碼重新生成 Bundle 檔案。
- **結果**：手動注入的 `_purify` 函數隨之被抹除。這證明了直接修改編譯產物缺乏**持久性 (Persistence)**。

---

## 第二章：理論升級——「雙層鎖定」架構 (Double-Lock Architecture)

為了根除失效問題，我們將防禦線從單一的「出口攔截」擴展為「生成端 $\rightarrow$ 傳輸端」的雙重鎖定。

### 2.1 第一層：認知 Hook (Cognitive Hook) —— Agent 端的自律
- **定義**：在 LLM 產生內容後，交付給 Gateway 之前的階段進行清洗。
- **實作**：將 `output_sanitizer.py` 納入 `AGENTS.md` 的最高級別【輸出鎖定協議】。
- **目的**：確保絕大部分內容在「出生」時就是純淨的。

### 2.2 第二層：傳輸 Hook (Transport Hook) —— Gateway 端的強權
- **定義**：在數據離開伺服器、進入網路傳輸前的最後一環進行攔截。
- **實作**：透過 `Gateway-Guardian` 腳本，在伺服器啟動時動態注入 `_purify` 引擎。
- **目的**：作為最後一道物理防線，攔截任何繞過認知 Hook 的洩漏符號。

---

## 第三章：實作落地與工程路徑

### 3.1 權限解鎖：打破 root 限制
由於 Gateway 位於 `/opt/homebrew`，普通腳本無法修改。我們實施了權限移交：
`sudo chown -R $(whoami) /opt/homebrew/lib/node_modules/openclaw/dist`
這使得攔截腳本能在無需 `sudo` 的情況下，在背景靜默地執行注入。

### 3.2 守護者腳本 (`gateway_guardian.py`) 的設計
守護者不再是單純的修改工具，而是一個**自檢引擎**：
1. **掃描**：定位目前的 `server.impl-*.js` 檔案。
2. **校驗**：檢查檔案中是否已存在 `_purify` 簽名。
3. **注入**：若發現簽名缺失（因更新而被覆蓋），立即重新注入純化引擎。

### 3.3 確定性執行循環
我們將此流程整合進系統的【絕對恢復協議】：
`啟動/恢復` $\rightarrow$ `觸發 Gateway-Guardian` $\rightarrow$ `物理注入` $\rightarrow$ `啟動服務`。

---

## 第四章：維護與復刻指南

若要在新環境復刻此系統，請遵循以下步驟：

1. **部署工具**：將 `output_sanitizer.py` 與 `gateway_guardian.py` 放置於 `/bin/` 目錄。
2. **權限移交**：執行 `chown` 指令解鎖 `dist` 目錄寫入權。
3. **配置協議**：將輸出鎖定協議寫入 `AGENTS.md`，將自檢指令寫入啟動恢復流程。
4. **激活攔截**：首次執行 `gateway_guardian.py` 並重啟 Gateway。

## 結論：從概率到確定的飛躍
本架構將「攔截」從一個依賴於 AI 自覺的概率行為，轉化為一個由程式強制的物理過程。
- **不依賴 AI 記憶** $\rightarrow$ 依賴物理檔案。
- **不依賴 AI 誠實** $\rightarrow$ 依賴傳輸攔截。
- **不依賴版本穩定** $\rightarrow$ 依賴啟動自檢。

這套系統確保了無論經歷多少次更新或還原，OpenClaw 的輸出將永遠保持物理級的純淨。
