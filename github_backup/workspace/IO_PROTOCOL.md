# 高延遲 I/O 處理協議 (High-Latency I/O Protocol)

本文件為系統執行行為之強制規範，旨在解決大容量目錄掃描導致的系統阻塞與 Token 浪費問題。

## 🛡️ 核心執行準則

### 1. 預判與分級 (Pre-emptive Grading)
- **規模判定**：在執行任何遍歷型命令（如 `du`, `find`, `grep -r`）前，必須先評估目標目錄之規模。
- **大容量禁令**：針對已知或預期的大容量目錄（如 `backups/`, `repo/`），**嚴禁**直接執行全量掃描。
- **優先路徑**：必須優先使用 `--max-depth=1` 或針對特定子目錄進行分段查詢，禁止一次性遍歷數 GB 等級的數據。

### 2. 輪詢終止機制 (Polling Termination)
- **三擊停止原則**：針對後台執行之沉重任務，單一命令若在 3 次輪詢 (`process poll`) 後仍未產出結果 → **必須立即終止輪詢**。
- **強制報備**：終止輪詢後，必須立即告知用戶：「任務正在後台執行，預計耗時較長，我將在有結果時立即通知」，禁止在沉默中反覆嘗試。
- **排他性執行**：在前一個高延遲 I/O 任務完成前，嚴禁啟動同類型的競爭性掃描任務。

### 3. 快捷路徑優先 (Fast-Path Priority)
- **輕量替代**：優先嘗試使用 `ls -lh` 等僅查詢第一層元數據的命令，而非直接調用 `du -sh` 遍歷所有子檔案。
- **快取優先**：若系統已有相關狀態紀錄，優先讀取記錄而非重新掃描。
