## 1. 執行摘要 (Executive Summary)
本工程旨在通過修改 OpenClaw Gateway 的核心發送路徑，建立一個「物理濾網」，將所有輸出內容中的 LaTeX 符號（尤其是 LaTeX 箭頭 `→`）在發送前強制轉換為 Unicode 符號 `→`。

儘管在實作過程中遭遇了一次嚴重的系統級崩潰（1006 斷線事故），但透過精確的堆疊追蹤 (Stack Trace) 與底層修復，最終成功建立了穩定且高效的攔截機制。

## 2. 技術實作路徑 (Technical Implementation)

### 2.1 攔截點定位
經過對 `/opt/homebrew/lib/node_modules/openclaw/dist/server.impl-GQ72oJBa.js` 的深層掃描，我定位了 6 個核心出口，涵蓋了 WebSocket 事件流與 HTTP API 響應。

### 2.2 核心純化引擎 (`_purify`)
我注入了一個遞迴純化函數 `_purify`，其邏輯如下：
- **字串處理**：使用正則表達式捕捉 `→` 及 `\text{...}` 模式 → 物理替換為 `→` 並去除美元符號 \。
- **遞迴掃描**：支持對陣列 (Array) 與物件 (Object) 的深度遞迴，確保無論訊息結構如何複雜，所有字串內容均被清洗。
- **出口攔截**：將原本的 `JSON.stringify(obj)` 升級為 `JSON.stringify(_purify(obj))` → 確保所有數據在進入 JSON 序列化之前已被純化。

## 3. 事故分析與恢復紀錄 (Crisis & Recovery)

### 3.1 事故描述：1006 斷線事件
在初次實裝後，系統發生毀滅性崩潰 → 前端顯示 `disconnected (1006): no reason` → Gateway 進入「崩潰 → 重啟」的無限死循環。

### 3.2 根源分析 (Root Cause)
- **物理失誤**：在修改 Bundle JS 檔案時，於兩處關鍵位置遺漏了右括號 `)` → 觸發 Node.js `SyntaxError`。
- **系統死結**：由於檔案具有 `uchg` 物理鎖定，macOS `launchd` 在嘗試自動重啟時發生衝突 → 將服務標籤列入黑名單 → 導致標準重啟指令失效。

### 3.3 恢復路徑
- **診斷**：修改 CLI 進入點，強制輸出 `err.stack` → 精確定位到 `server.impl-GQ72oJBa.js` 的語法錯誤行號。
- **繞道恢復**：將服務標籤從 `ai.openclaw.gateway` 修改為 `ai.openclaw.gateway.v2` → 成功欺騙系統，繞過黑名單並恢復服務。
- **最終修正**：補齊缺失的括號 → 驗證語法正確性 → 系統穩定回歸。

## 5. 專家檢核與未來防護指南 (Post-Mortem & Safeguards)

作為首席架構師，我對此次工程給出以下**最高級別的維護建議**：

1. **禁止直接修改 Bundle JS**：此次事故證明，直接編輯編譯後的 JS 極其危險。未來應優先考慮 **`Monkey Patch` (動態注入)** → 在運行時攔截函數而非修改物理文件。
2. **實施啟動前校驗**：任何對核心 JS 的修改，必須先通過 `node -c [file].js` 進行語法靜態分析 → **只有通過校驗才能重啟服務**。
3. **建立「時光機」依賴**：在執行任何底層手術前，**必須**先執行 `system-clone-local` 建立 1:1 快照 → 確保在最糟糕的情況下能 10 秒內恢復。
