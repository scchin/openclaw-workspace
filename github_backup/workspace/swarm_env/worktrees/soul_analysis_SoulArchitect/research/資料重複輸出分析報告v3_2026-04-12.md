## 二、方案A 實施：技能層防翻譯

### 2.1 發現：評論被截斷的根本原因

觀察用戶截圖中的「一字排開」慘狀（評論被截成 16 行），原因是：

`run.py` 中用 `──` 分隔店家，每家店的最後一行是 `🗺️ 地圖：...`，但中間夾雜了大量被翻譯截斷的評論。這不是技能代碼的問題，而是 **Qclaw 對原始文字進行了翻譯志願者處理後，再以某种方式呈現**。

### 2.2 方案A 具體實施：標記為已處理

在 `format_full_output()` 的所有輸出行前加上語言標記 `[已處理]`，防止 Qclaw 識別為需要翻譯的內容：

```python
# 在 format_full_output 開始處加入防呆標記
# 讓系統知道這些內容已處理過，不需要翻譯
```

### 2.3 方案B：控制 Qclaw 應用程式設定

在 Qclaw 的 Preferences 中：
- **General → Language**：確認已設為「繁體中文（台灣）」，避免自動翻譯簡體/英文
- **Chat → Translation**：檢查是否有「自動翻譯收到的訊息」選項（路徑：Qclaw → Preferences → Chat → Translation）

## 三、發現：評論被截斷成 16 行的原因

用戶截圖顯示的「一字排開」慘狀，經分析後發現：

**`run.py` 中 `_wrap_chars` 和 `_wrap_line` 的邏輯是：**
- 對長無空格的英文句子（> 50 字元且無空格/標點），每次遞迴時把所有剩餘文字當成"一個單字"
- 導致 50 字元的句子被截成 16 行（每次只取一個字）

**這是技能代碼的 bug，而非 Qclaw 的問題。**

### 3.1 根本修復：改善 `_wrap_chars` 的邏輯

當剩餘文字長度小於 `width` 時，直接輸出，不要繼續截斷。
