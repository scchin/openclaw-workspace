# MemPalace Phase 4 Hybrid 基線測試報告

## 1. 測試對象
- **版本**：MemPalace v3.3.0 (Phase 4 Hybrid)
- **狀態**：本地穩定運行中

## 2. 六維度分析結果

| 維度 | 測試結果 | 詳細分析 |
| :--- | :--- | :--- |
| **搜尋引擎** | Hybrid (向量+關鍵字) | 採用 ChromaDB 向量檢索與傳統關鍵字匹配的混合模式，能處理語義查詢與精確匹配。 |
| **檢索率 (R@10)** | ~0.7 - 0.9 (估計) | 根據對現有記憶庫的隨機抽樣驗證，對明確事實的檢索率較高，但對跨文檔的模糊關聯檢索存在一定的隨機性。 |
| **記憶結構** | 分層樹狀 (Hierarchical) | 結構為：Wings $\rightarrow$ Halls $\rightarrow$ Drawers。這種結構提供了良好的組織感，但對於非結構化知識的快速插入有較高依賴。 |
| **後端支援** | ChromaDB + SQLite | 使用 ChromaDB 處理向量索引，SQLite 處理元數據與三元組，確保了本地部署的輕量化與高效能。 |
| **MCP 工具** | 標準記憶集 | 提供基本的 `search`, `mine`, `compress` 等 MCP 接口，能與 LLM 良好集成。 |
| **知識圖譜** | 三元組模型 (Triples) | 實作簡單的 (Subject, Predicate, Object) 三元組儲存，支持實體間的直接關聯，但缺乏複雜的圖算法支持。 |

## 3. 綜合評價
Phase 4 Hybrid 版本已實現基礎的「記憶宮殿」功能，能有效將碎片化信息轉化為可檢索的知識庫。其核心優勢在於本地化與混合檢索，但缺點在於結構維護成本較高，且知識圖譜的深度挖掘能力有限。

## 4. 結論
該版本提供了一個穩定的基線，為接下來升級至 Hybrid v5 提供對比基準。
