# 女媧系統與虛擬專家工作小組 (DED) 整合使用指南

本指南旨在詳細說明 **女媧 (`huashu-nuwa`)** 技能與 **虛擬專家工作小組 (`DED - Dynamic Expert Dispatcher`)** 之間的協作邏輯。這套系統將「認知提煉」與「即時調度」相結合，讓 AI 能在不同專家視角之間快速切換，提供高純度的專業分析。

---

## 第一章：虛擬專家工作小組 (DED) 核心架構

### 1.1 系統定義
「虛擬專家工作小組」並非單一程式，而是一套由三個核心組件構成的協作機制：
* **`nuwa-dispatcher` (調度員)**：扮演「大腦」角色，負責意圖識別與認知分流。
* **`EXPERT_REGISTRY.md` (能力圖譜)**：扮演「索引」角色，記錄所有可用專家的認知 DNA。
* **`huashu-nuwa` (執行引擎)**：扮演「身體」角色，負責根據注入的 DNA 執行具體的分析輸出。

只要這三個組件配置完成，系統即處於隨時待命狀態。

### 1.2 即時認知分流流程
當使用者提問時，系統會啟動**「即時認知分流」**模式：
1. **意圖偵測**：調度員掃描輸入，識別觸發詞（如：「專家的看法」、「專家分析」等）。
2. **即時匹配**：調度員調用 `scripts/dispatch.py` 掃描 `EXPERT_REGISTRY.md`，根據問題領域與思考風格，毫秒級篩選出最合適的「專家原型」。
3. **DNA 注入**：將選定專家的「認知 DNA」（思考風格、邏輯模式）即時注入 `huashu-nuwa` 引擎，使其轉化為該專家身份進行回答。

**設計核心**：追求**「自包含 (Self-contained)」**。每個專家目錄不僅包含 `SKILL.md` 認知框架，還包含所有调研證據 (`references/`)，確保技能可獨立複製分發，無需依賴外部數據庫。

### 2.2 從蒸餾到固化的過程
女媧在「造人」過程中採取物理與邏輯雙軌制：
* **物理存儲 (認知固化)**：通過生成詳細的 `SKILL.md` 文件，將心智模型、表達 DNA 與誠實邊界永久化。
* **邏輯登記 (發現機制)**：將專家的基本定義（名稱、專長、原型、模式）登記至全局索引文件 `EXPERT_REGISTRY.md` 中。

## 第三章：系統聯動與溝通模式

### 3.1 聯動關係：大腦 → 索引 → 身體
女媧系統與 DED 之間形成了一種強依賴的共生關係：
**`nuwa-dispatcher` (調度員/大腦)** → **`EXPERT_REGISTRY.md` (能力圖譜/索引)** → **`huashu-nuwa` (執行引擎/身體)**

4. **身份注入** → 將 DNA 作為高權重指令注入 `huashu-nuwa`。
5. **高純度輸出** → `huashu-nuwa` 切換推理模式，生成專家級分析。
