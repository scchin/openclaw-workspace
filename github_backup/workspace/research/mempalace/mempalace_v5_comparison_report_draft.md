# MemPalace Hybrid v4 vs v5 差異分析報告

## 1. 測試概況
本報告對比了 MemPalace 的兩個主要階段版本：**Phase 4 Hybrid (v3.3.0)** 與 **Hybrid v5 (Develop Branch)**。測試聚焦於檢索能力、結構靈活性及工具集成度。

## 2. 核心維度對比表

| 維度 | Phase 4 Hybrid (v3.3.0) | Hybrid v5 (Develop) | 差異判定 |
| :--- | :--- | :--- | :--- |
| **搜尋引擎** | 基礎 Hybrid (向量 + 關鍵字) | 進階 Hybrid (加入時序、偏好權重提升) | **顯著提升** → 檢索精度更高 |
| **檢索率 (R@10)** | ~0.7 - 0.9 (估計) | **88.9% (LoCoMo 官方)** | **量化提升** → 穩定性增強 |
| **記憶結構** | 靜態分層 (Wings > Halls > Drawers) | 動態對象分層 (Person/Project → Room) | **結構優化** → 更符合人類記憶邏輯 |
| **後端支援** | 固定 ChromaDB + SQLite | 插件式後端 (Pluggable Backends) | **擴展性提升** → 支援多種向量庫 |
| **MCP 工具** | 標準記憶工具集 | 全量 MCP 工具集 (29 個工具) | **功能爆發** → 支援圖查詢、日記等 |
| **知識圖譜** | 簡單三元組 (Triples) | 時序實體關係圖 (Temporal ER Graph) | **能力飛躍** → 支援效期與時間線 |

## 3. 深度分析

### 3.1 檢索能力的演進
Hybrid v5 不再僅僅依賴向量相似度，而是引入了**時序接近度 (Temporal-proximity boosting)** 與**偏好模式提取**。這解決了 Phase 4 中常見的「雖然語義接近但時間久遠」或「關鍵字匹配但不符合當前上下文」的問題，使得 R@10 達到工業級水準。

### 3.2 從「儲存庫」到「知識圖譜」
Phase 4 的知識圖譜僅能記錄簡單的 A → B 關係。v5 引入了**有效期限 (Validity Windows)**，能夠記錄「A 在某時間段內是 B 的主管」這種動態資訊，這對於追蹤複雜的項目變更或人際關係至關重要。

### 3.3 MCP 生態的成熟
v5 提供的 29 個 MCP 工具將 MemPalace 從一個「檢索插件」提升為一個「記憶操作系統」。 agent 現在可以直接管理日記、執行跨 Wing 導航以及精確控制 Drawer 的生命週期。

## 4. 最終結論
**Hybrid v5 是一次底層邏輯的重構，而非單純的功能疊加。** 
它將 MemPalace 從一個高效的 RAG 工具轉化為一個真正的 **本地 AI 記憶操作系統**。對於需要處理大量動態資訊、複雜實體關係以及高精度檢索的場景，v5 具有壓倒性優勢。
