# 🤝 技術審查會議紀錄 (Integration Notes)
**日期**: 2026-04-25
**參與者**: Project Lead, OSINT Expert, NLP Expert, Market Research Architect

## 1. 整合路徑圖 (Integrated Workflow)
`數據採集 (Expert A)` → `量化分析 (Expert B)` → `商業驗證 (Expert C)` → `執行提案 (Final)`

### 詳細流程分解：
1. **採集端**: 採用「混合模式」。針對 Dcard 等低門檻平台全量抓取；針對 TikTok/IG 採取「定向抽樣」 (Targeted Sampling)，以降低封號風險。
2. **分析端**: 數據進入 `Preprocessing` → `BERTopic` 定義話題 → `RoBERTa-ABSA` 提取具體產品維度的正負面情緒。
3. **轉化端**: 將 ABSA 的結果填入 `優先級矩陣` → 找出「高影響力-低滿意度」的痛點 → 作為問卷設計的基石。
4. **驗證端**: 建立 `社群發現 → 定向問卷 → 需求定論` 的閉環。

## 2. 關鍵衝突與解決方案
- **衝突**: 數據規模需求 vs. 反爬蟲限制。
- **對策**: 引入「兩段式採集法」。
  - **第一階段**: 小規模抽樣 → 用於快速迭代分析模型。
  - **第二階段**: 確定核心維度後 → 精準採集相關關鍵字數據，減少不必要的流量。

- **衝突**: 文本噪聲 → 分析精度。
- **對策**: 在 `NLP Pipeline` 中增加 `LLM-based Cleaning` 步驟，利用 GPT-4o 等模型對小部分數據進行「金標籤」校準，再擴展至全量數據。

## 3. 最終提案核心要件 (Core Requirements for Proposal)
- 必須包含一個 **「物理可行性清單」** (工具鏈、代理成本、預期時程)。
- 必須包含一個 **「需求驗證模板」** (如何從分析結果轉化為問卷題目)。
- 必須包含 **「風險緩衝方案」** (如 API 失效後的替代路徑)。
