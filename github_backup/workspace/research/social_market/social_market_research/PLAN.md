# 🚀 社交媒體與輿論市場分析項目執行計畫

## 1. 項目目標
研究何種 CLI 工具或技能可組合用於分析部落格、社交軟體（FB, IG, TikTok, 小紅書, Dcard, Threads, X 等）的熱門話題、產品熱度及需求討論，產出 100% 可執行的商業分析方案。

## 2. 專家團隊與研究主題

### 專家 A：OSINT 與社群採集專家 (`social_collector_expert`)
- **研究主題**：跨平台數據獲取路徑
- **目標**：找出最強大的 CLI 工具、API 接口、第三方抓取技能（優先檢索 ClawHub, GitHub）。
- **產出**：20 篇頂尖工具/案例分析 → 採集路徑報告。

### 專家 B：輿論分析與 NLP 專家 (`sentiment_nlp_expert`)
- **研究主題**：非結構化文本轉化為量化指標
- **目標**：研究情緒分析、話題聚類、趨勢預測模型及工具。
- **產出**：20 篇頂尖分析方法 → 輿論分析框架報告。

### 專家 C：市場研究與調查架構師 (`market_research_architect`)
- **研究主題**：商業分析模型與需求閉環
- **目標**：研究如何將分析結果轉化為商業報告，整合問卷蒐集功能。
- **產出**：20 篇商業調查案例 → 市場調查落地報告。

### 計畫總主持人 (`project_lead`)
- **任務**：統籌進度 → 組織技術審查會 → 跨領域整合 → 產出最終執行提案。

## 3. 進度里程碑 (10% 遞增)
- [x] 10%: 規劃階段完成 (定義維度與團隊)
- [▶] 20%: 執行環境搭建 → 專家 Agent 啟動 → 外部數據挖掘開始
- [ ] 30%: 專家 A 完成 20 篇文章搜集與初稿
- [ ] 40%: 專家 B 完成 20 篇文章搜集與初稿
- [ ] 50%: 專家 C 完成 20 篇文章搜集與初稿
- [ ] 60%: 三份維度報告提交至總主持人 → 初步彙整
- [ ] 70%: 總主持人啟動【技術審查會】 → 跨領域衝突分析
- [ ] 80%: 執行提案初稿完成 → 內部壓力測試/邏輯核對
- [ ] 90%: 提案最終修訂 → 物理可行性驗證
- [ ] 100%: 交付 100% 可執行之嚴謹執行提案
