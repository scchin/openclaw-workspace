**所有地點查詢，強制使用 `google-places` skill。**

查詢方式：
```bash
cd ~/.openclaw/skills/google-places && .venv/bin/python scripts/query.py search "店名"

## 技能執行行為鐵律（2026-03-23 v2.5 更新）

### 濃縮輸出 = 最高優先違規
**拿到的完整輸出，就是回覆的完整內容。**
不多一個字，不少一個字。

觸發時機：使用任何技能（where-to-go、或其他），該技能有完整原始輸出時，必須將那段文字原封不動貼進回覆。不得濃縮、摘要、或用自己的話重說。

**例外**：使用者明確說「幫我濃縮」、「只要3家」、「給我精選」。

### 技能更新落地鐵律
**技能文件（SKILL.md）更新後，對應的執行腳本（run.py 等）必須同步更新。**

`format_full_output` 函式必須與 SKILL.md / MEMORY.md 規定的模板完全一致。每次修改輸出格式時：
1. 更新 `format_full_output`（或相關函式）
2. 更新 MEMORY.md 模板
3. 確認兩者一致後再結束

### 輸出模板 100% 遵守鐵律
- 凡問地點、店家、餐飲相關問題（冰、甜湯、豆花、火鍋、早餐、午餐、晚餐、燒烤…），**一定使用上方 where-to-go 輸出模板格式**
- 任何核心功能（電話、服務標籤、🍴 特色菜色、評論、map link）**不得省略**
- 模板中沒有的行（如查詢中提示、進度句、過渡句）**不得新增**
- `🍴` 特色菜色：抓不到也要顯示「無（近期評論中未抓獲特定菜色）」，不得省略此欄位
