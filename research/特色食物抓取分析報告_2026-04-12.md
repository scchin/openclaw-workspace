# 店家「🍴 特色」食物抓取方法分析報告

**日期：** 2026-04-12  
**研究者：** King Sean of KS  
**目標：** 研究如何正確有效抓取 Google Maps 店家頁面中的「特色食品 / 特色菜色」欄位

---

## 一、目前實作現況

### 1.1 現有技能與架構

| 技能 | 負責範圍 | 特色抓取方式 |
|------|---------|------------|
| `google-places` | 單一店家完整查詢（CDP + API） | Phase 2 點擊「評論」tab → 抓導航按鈕文字 + 熱門品項（`[role=radio]`） |
| `where-to-go` | 餐廳列表搜尋 + 評論分析 | `_dish_from_reviews()` 掃描評論文字，正則比對 `_DISH_PATTERNS`（200+ 條目） |

### 1.2 目前 CDP 抓法（google-places / query.py）

點擊「評論」tab 後，執行 JS 抓取兩種資料：

**① 評論區上方的食物分類導航按鈕**
```javascript
var els = document.querySelectorAll('[role=navigation] button,[role=tablist] button,[role=tab]');
// 過濾：長度 1-10 字、不含「X則」數字
// 回傳：food_categories（陣列，最多 15 個）
```
→ 對應用戶截圖中「評論」標籤上方的食物分類（如：漢堡、早午餐、咖啡、甜點等）

**② 評論摘要區的熱門品項**
```javascript
var radios = document.querySelectorAll('[role=radio]');
// 取 aria-label 含 "mentioned in" 的項目
// 或 div 文字含\n分隔的兩段（標題+則數）
```
→ 對應 Google Maps 的「大家提到」區塊

### 1.3 目前評論正則抓法（where-to-go / run.py）

```python
_dish_from_reviews(reviews)
```
- 對每一則評論內容遍歷 `_DISH_PATTERNS`（200+ 種菜色關鍵字）
- 找到後擷取關鍵字後 60 字作為摘要
- 最多取 5 項，用 `dict.fromkeys()` 去重保持順序

## 二、網路研究與技術文獻

### 2.1 Google 官方 API 方案

| 方法 | 說明 | 可用性 |
|------|------|--------|
| **Google Places API (New) `includeBusinessMenus`** | 可取回結構化菜單資料（dish names、descriptions、prices） | ✅ 官方支援，收費 |
| **`reviewSummary` field mask** | AI 生成的評論摘要，可能提及熱門菜色 | ✅ 官方支援，收費 |
| **`dishes` 欄位** | Place Details 回傳的菜色推薦 | ✅ 官方支援 |

> 參考：https://developers.google.com/maps/documentation/places/web-service/review-summaries  
> 參考：https://www.foodspark.io/fetch-restaurant-menus-using-google-api/

### 2.2 第三方爬蟲工具

| 工具 | 特色 | 风险 |
|------|------|------|
| **Outscraper** | 支援大量評論抓取、菜單連結 | 付費、需遵守 Google ToS |
| **Apify Google Maps Scraper** | 可抓 popular dishes、菜單 | 需付費、有反爬機制 |
| **Bright Data CDP Browser** | 真實瀏覽器控制，繞過反爬 | 昂貴、商用 |
| **Octoparse** | 無程式碼視覺化爬蟲 | 對 Google Maps 效果有限 |

> 參考：https://outscraper.com/google-maps-reviews-api/  
> 參考：https://www.apify.com/gtgyani206/google-maps-restaurant-scraper

### 2.3 法律合規性

- **hiQ v. LinkedIn 先例**：公開資料網頁爬蟲在美國法律下被視為合法
- **Google ToS 禁止爬蟲**：屬於契約違規，非刑事罪
- **風險**：IP 封鎖、API Key 被撤、帳號停用
- **建議**：以 Google 官方 API 為主，CDP 為輔

> 參考：https://scrap.io/scrape-google-gaps-legal

## 三、目前方法的有效性分析

### 3.1 CDP food_categories 方法

**優點：**
- 直接取得 Google 官方分類標籤（使用者看到的即是如此）
- 不依賴評論文字，適用新店家或評論少的店

**缺點：**
- 依賴 DOM 結構，Google Maps 更新 UI 可能失效
- 只取得到「分類名稱」（如「咖啡」「甜點」），不是具體菜色（如「拿鐵」「提拉米蘇」）
- 查詢器長度限制 `t.length <= 10` 可能漏掉長分類名

**測試指令：**
```bash
# 對晨家早午餐進行 CDP 完整查詢
cd ~/.openclaw/skills/google-places
python3 -c "
import asyncio, json, websockets, subprocess, os, time

async def test():
    # 先確保瀏覽器已啟動
    import urllib.request
    try:
        req = urllib.request.Request('http://127.0.0.1:18800/json')
        with urllib.request.urlopen(req, timeout=3) as r:
            tabs = json.loads(r.read())
            ws_url = tabs[0]['webSocketDebuggerUrl']
    except:
        print('Browser not running')
        return
    
    # 直接對晨家早午餐 CDP
    from scripts.query import scrape_price_from_browser
    result = await scrape_price_from_browser('ChIJs9ZwdXQXaTQRa5LL3bcnLq4')
    print('food_categories:', result.get('food_categories'))
    print('popular_items:', result.get('popular_items'))

asyncio.run(test())
"
```

### 3.2 評論正則比對方法

**優點：**
- 不受 UI 變化影響，文字比 DOM 穩定
- 可以抓到具體菜色與評論片段

**缺點：**
- 需要足夠多的評論才有效（依賴 6M→1Y→2Y→全部遞進）
- 評論中沒提到特色菜就會顯示「無」
- 依賴 `_DISH_PATTERNS` 的覆蓋範圍

**優化方向：**
- 擴充 `_DISH_PATTERNS`，尤其是早餐類（目前已有陽春麵、蛋餅、蘿蔔糕等）
- 加入更多形容詞+蛋白質組合（如「明太子」「胡麻」「剝皮辣椒」）

## 五、改進建議

### 5.1 短期（已可執行）

1. **合併 CDP food_categories + 評論正則**
   - CDP food_categories 做為主要特色（Google 官方分類）
   - 評論正則做為 fallback（當 CDP 抓不到時補充具體菜色）

2. **擴充 `_DISH_PATTERNS`**
   - 對照晨家早午餐的評論與截圖中的分類，確認覆蓋完整
   - 加入早餐店常見品項：蛋餅皮、鍋貼、水煎包、餡餅、蔥油餅、蘿蔔糕、碗粿、肉粽、飯糰、蚵仔麵線、麵線、豆漿、米漿、鹹豆漿、甜豆漿、燒餅、油條、小籠湯包、水餃、餛飩麵、餛飩湯、肉圓、蚵仔煎、肉燥飯、滷肉飯、雞腿飯、便當、自助餐、素食、蔬食等

3. **提高 CDP 食物分類抓取精確度**
   - 放寬長度限制（目前 `t.length <= 10` 可能漏掉「義式料理」等）
   - 加入 `[role=region]` 中含食物相關關鍵字的文字

### 5.2 中期（需要研究）

4. **整合 Google Places API New 的 `dishes` 欄位**
   - 在 `google-places` 的 `get_details_json()` 中加入 `dishes` 欄位
   - 直接取得 Google 官方結構化菜色資料

5. **reviewSummary 欄位**
   - 在 `format_output()` 中加入 `review_summary` 的呈現

### 5.3 長期（需商業合作）

6. **使用第三方專業爬蟲服務**
   - 如 Outscraper、Apify 取得更完整的評論資料
   - 適合大量店家分析
