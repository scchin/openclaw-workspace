### 重要實作細節

**營業狀態邏輯（修正：只用系統時間 + hours資料）：**
- `datetime.now()` 取得系統時間（亞洲/台北時區，CST，GMT+8）
- 只用 `goplaces hours` 欄位中的週五營業時間計算
- 不用 `open_now`（Google API 有時區/更新延遲問題）
- 往前解析關門時間：regex 只取 `–` 之後的時段（PM）作為關門時間
- 跨日夜處理：開門時間 < 關門時間則跨日（+1 day）
- 若營業中但已過關門時間：顯示「已超過關門時間」
- 若今日休息：找下一個開門日

**CDP 架構：**
- WebSocket URL：`ws://127.0.0.1:18800/devtools/page/<tab_id>`
- 分頁探索：GET `http://127.0.0.1:18800/json` 取 `webSocketDebuggerUrl`
- 目標分頁：URL 含 `maps` + `place`
- JS 執行：同步模式（`awaitPromise: False`）
- 讀取每人消費：regex `$1-200 per person Reported by 81 people`
- 讀取店家備注：按鈕 `Serves/takeaway/delivery` + `Located in:` + `Photo of:` + 熱門品項（role=radio）

**依賴：**
- `pip3 install --break-system-packages websockets`
- OpenClaw 瀏覽器分頁須開啟 Google Maps 店家頁面

**已知限制：**
- CDP 需要 OpenClaw 瀏覽器處於正確 Google Maps 分頁
- "每人消費" 並非所有店家都有
